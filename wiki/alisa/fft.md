# Что такое FFT‑анализ звука

FFT (Fast Fourier Transform) — это быстрое преобразование Фурье, алгоритм, который позволяет эффективно выполнять дискретное преобразование Фурье (ДПФ). В контексте анализа звука FFT преобразует сигнал из временной области (амплитуда звука в зависимости от времени) в частотную область (какие частоты присутствуют в сигнале и с какой амплитудой).

## Как это работает

### Исходный сигнал. 
Звуковой файл (например, .wav) представляет собой последовательность отсчётов амплитуды — график зависимости амплитуды от времени.

### Разбиение на блоки. 
FFT работает с блоками данных фиксированного размера (так называемым FFT size или размером окна). Типичные размеры — степени двойки: 128, 256, 512, 1024, 2048 и т. д.

### Преобразование. 
Алгоритм FFT раскладывает каждый блок на набор синусоидальных волн разных частот. На выходе получается массив комплексных чисел, из которых можно извлечь:

- [x] Амплитуду каждой частотной составляющей.
- [x] Фазу каждой составляющей.

### Результат. 
Визуально результат часто представляют в виде спектрограммы или графика спектра: по горизонтальной оси — частота (Гц), по вертикальной — амплитуда (дБ).

### Ключевые понятия

- FFT size (размер окна). 
- Количество отсчётов в обрабатываемом блоке. 
Чем больше размер окна:
- тем выше частотное разрешение (можно точнее определить частоту);
- тем хуже временное разрешение (усреднение происходит за более длинный отрезок времени).

- Бины (bins). 
Частотные «ячейки» или полосы, на которые делится диапазон частот. Количество бинов равно половине размера окна FFT (для реального сигнала). Например, при FFT size = 1024 будет 512 бинов.

- Ширина бина. Определяется частотой дискретизации (fs) и размером окна:

ширина бина = FFT size fs
​
Например, при fs = 44,1 кГц и FFT size = 1024 ширина бина ≈ 43 Гц.

### Весовые окна (windows). 
Для уменьшения артефактов на границах блока (утечки спектра) сигнал перед FFT умножают на весовую функцию (окна Ханна, Блэкмана и др.).

### Усреднение спектров. 
Для снижения влияния шума и получения более гладкой картины усредняют результаты нескольких последовательных FFT.

### Что можно определить с помощью FFT‑анализа
- [x] Частотный состав звука. Какие частоты присутствуют и насколько они сильны (например, выделить ноты в музыкальном аккорде).
- [x] Гармонические искажения. Побочные частоты, кратные основной (характерно для перегруженных усилителей или динамиков).
- [x] Уровень шума. Оценить уровень фонового шума или шипения.
- [x] Соотношение сигнал‑шум (SNR). Сравнить уровень полезного сигнала с уровнем шума.
- [x] Межмодуляционные искажения. Появление частот, которых не было в исходном сигнале (комбинации частот).
- [x] Перекрёстные помехи (crosstalk). Проникновение сигнала из одного канала в другой.
- [x] Анализ резонансов. Найти частоты, на которых резонирует помещение или корпус инструмента.

### Где применяется
- Аудиотехника. Тестирование усилителей, микрофонов, динамиков, ЦАП/АЦП.
- Звукорежиссура. Настройка эквалайзера, поиск проблемных частот, удаление гула (50/60 Гц).
- Распознавание речи. Выделение формант (характерных частот гласных звуков).
- Музыкальная информатика. Определение высоты тона, анализ тембра инструментов.
- Диагностика. Анализ шумов механизмов для выявления дефектов.

Получаем 512 частотных бинов.

Ширина каждого бина: 1024 44100 ≈ 43,07 Гц.

### Оси графика:

горизонтальная ось (X) — время;
вертикальная ось (Y) — частота в герцах (Гц). 
Амплитуда (громкость) отображается через интенсивность цвета или яркости:

более яркие или красные области — сильные сигналы (высокая амплитуда);
тёмные или синие области — слабые сигналы (низкая амплитуда). 
Некоторые особенности интерпретации:

горизонтальные линии указывают на стабильные частоты, например, основной тон инструмента или голоса;
вертикальные импульсы могут соответствовать резким звукам (например, ударным инструментам);

равномерно распределённые гармоники могут указывать на низкочастотный гул;

Их можно создавать с помощью аудиоредакторов (например, Audacity, Spectrogram, Ocenaudio) или специализированных библиотек для программирования (например, librosa в Python).

Какие есть библиотеки для создания спектограм на javascript? Также интересно есть ли какие то библиотеки для создания fft анализов?
Алиса
Для создания спектрограмм и выполнения FFT-анализа в JavaScript существуют несколько библиотек, которые предоставляют необходимые инструменты.

Библиотеки для создания спектрограмм
Spectroplot-js. Библиотека для визуализации спектрограмм на HTML-холсте. Поддерживает различные типы данных, включая аудиофайлы (WAV, MP3 и др.) и необработанные SDR-данные. Предоставляет возможности выбора оконных функций (Бартлетт, Хэмминг, Ханн и др.), цветовых карт и масштабирования. Есть функционал для отображения частотной и временной шкалы, а также информации о дБ. 

Spectrogram-2v01.js. Библиотека для создания спектрограмм в веб-приложениях. Позволяет создавать отображения типа «waterfall» (данные «стекают» вниз) и «rasterscan» (растровое сканирование). Поддерживает отрисовку данных в реальном времени и предоставляет API для управления отображением. 

LightningChart JS. Библиотека для создания интерактивных графиков, включая 2D и 3D спектрограммы. Позволяет создавать приложения для анализа вибраций и звуковых волн. Поддерживает настройку цветовых палитр, каналов и других параметров. 

Библиотеки для FFT-анализа
jsfft. Небольшая и эффективная реализация быстрого преобразования Фурье (FFT) для Node.js и браузера. Работает с комплексными массивами и предоставляет методы для прямого (FFT) и обратного (InvFFT) преобразования. Подходит для обработки аудио- и других сигналов. 

fft.js. Реализация FFT с использованием алгоритма Radix-4. Поддерживает как комплексные, так и вещественные входные данные. Предоставляет методы для прямого преобразования (transform), обратного преобразования (inverseTransform), а также для работы с вещественными данными (realTransform). 

fft-js. Чистая реализация FFT на JavaScript (метод Кули-Тьюки). Включает функции для прямого и обратного преобразования, а также дополнительные утилиты для работы с частотами и амплитудами. Подходит для образовательных целей и простых задач. 

TensorFlow.js. Библиотека для машинного обучения, которая также предоставляет функцию tf.spectral.fft() для выполнения FFT. Работает с тензорами и позволяет вычислять одномерное дискретное преобразование Фурье по внутреннему измерению входных данных.

Обнаружение дрона по звуку возможно благодаря характерным акустическим особенностям, которые создают его двигатели, пропеллеры и аэродинамические эффекты. Для этого используются как слуховой анализ, так и технические методы с применением спектроанализа и машинного обучения. 

Характерные звуки дронов
Звук дрона зависит от его типа и конструкции:

FPV-дроны («камикадзе») издают резкий «мушино-комариный» звук. 
Дроны типа «Мавик» звучат похоже на работу стоматологической бормашины. 
Тяжёлые мультироторные дроны (например, «Баба-Яга») издают звук, напоминающий многократно усиленное жужжание майского жука. Если дрон гружёный, звук становится более низким, похожим на работу мопеда или газонокосилки. 
БПЛА самолётного типа с двухтактным бензиновым двигателем издают звук, похожий на отдалённый звук мопеда или газонокосилки. 
В общем случае акустический профиль дрона формируется несколькими источниками:

Пропеллеры. Вращение винтов создаёт периодические возмущения воздуха, которые слышны как низкочастотный гул. Частота основного тона пропеллера связана с его скоростью вращения и количеством лопастей. 
Электродвигатели. Бесколлекторные электродвигатели создают характерный высокочастотный «писк» в диапазоне 5–15 кГц. 
Аэродинамический шум. Воздушный поток вокруг корпуса и шасси дрона производит турбулентные шумы, похожие на «шипение» или «свист». 
Резонанс корпуса. Некоторые модели дронов обладают корпусами и рамами, которые усиливают определённые частоты, что приводит к появлению ярко выраженных резонансных пиков в спектре. 
Как отличить звук дрона от других источников шума
Чтобы обнаружить дрон среди фонового шума, нужно учитывать следующие особенности:

Повторяемость структуры. Фоновый шум (ветер, машины, голоса) чаще всего хаотичен, а у дрона есть регулярный спектр с гармониками на равных интервалах. 
Смещение частот при манёврах. При ускорении или торможении дрон изменяет обороты пропеллеров, а значит, частота основного тона сдвигается (эффект Доплера). 
Отсутствие природных аналогов. Большинство дронов создают спектры с несколькими выраженными пиками на узких частотах, что редко встречается в природе. 
Технические методы обнаружения
Для автоматического обнаружения дронов по звуку используются различные технологии:

Фурье-анализ (FFT). Прямое преобразование Фурье позволяет определить, на каких частотах сосредоточена энергия звука. Пики на фиксированных частотах указывают на наличие БПЛА. 
Машинное обучение. Нейросети обучаются на базе аудиозаписей и могут отличать квадрокоптер от других источников шума с высокой точностью. 
Корреляция сигнатур. Каждая модель дрона имеет уникальный «отпечаток» спектра. Система сопоставляет текущий сигнал с базой известных профилей. 
Существуют готовые решения и проекты, реализующие такие методы:

UAVSoundAnalysis — набор инструментов на Python для анализа звука дронов, включающий предобработку, машинное обучение и обнаружение акустических сигнатур. 
UAV-Detector — проект на GitHub, демонстрирующий систему акустического обнаружения дронов с использованием Mel-спектрограмм и свёрточных нейронных сетей. 
Система от СГТУ. Гибридный подход, сочетающий частотно-временной анализ звука с алгоритмами глубокого обучения. Система способна работать в условиях городского шума.

Коммерческие комплексы (например, «МСБ — Дрон детектор»), которые используют нейросетевую обработку аудиосигналов и могут обнаруживать дроны на расстоянии до 500–1000 м. 
Ограничения акустического метода
У акустического метода есть ряд ограничений:

Дальность. Звук быстро затухает, особенно высокочастотные компоненты. 
Фоновые помехи. Шум города, ветер, животные могут скрывать сигнатуру дрона. 
Множественные источники. Несколько дронов создают интерференцию, что усложняет анализ. 
Акустические методы часто используются в сочетании с другими технологиями (визуальным наблюдением, радиолокацией) для повышения эффективности обнаружения.

FFT (быстрое преобразование Фурье) в контексте обнаружения дронов по звуку — это метод анализа аудиосигнала, который позволяет разложить его на частотные составляющие. Это помогает выявить характерные особенности звука дрона, которые отличают его от фонового шума. 

Принцип работы FFT
FFT преобразует временной сигнал (например, запись звука с микрофона) в частотный спектр. То есть вместо графика, где по оси X отложено время, а по оси Y — амплитуда звука, получается график, где по оси X отложены частоты, а по оси Y — амплитуда этих частот. Это позволяет увидеть, какие частоты присутствуют в звуке и с какой интенсивностью.

В случае с дронами FFT помогает выявить:

Базовую частоту вращения пропеллеров. Она связана с количеством лопастей и скоростью вращения. Например, если пропеллер имеет 2 лопасти и вращается со скоростью 5000 об/мин, базовая частота может составлять около 100 Гц (с учётом того, что 1 об/мин = 1/60 Гц, а частота умножается на количество лопастей). 
Гармоники — кратные частоты базовой частоты. В спектре дрона часто наблюдаются несколько гармоник, расположенных на равных интервалах.

Особенности, связанные с аэродинамикой и конструкцией дрона. Например, турбулентность на лопастях может создавать высокочастотный «шип» (4–10 кГц), а выхлоп двигателя — низко-среднечастотный шум. 
Изменения в спектре при манёврах. При ускорении, торможении или изменении высоты полёта дрон меняет обороты пропеллеров, что приводит к сдвигу частот в спектре (эффект Доплера).

Оптимальная частота преобразования в частотный спектр
Для обнаружения дронов по звуку с помощью FFT рекомендуется обновлять спектр 5–20 раз в секунду (интервал 50–200 мс между преобразованиями). Разберу, почему именно так.

Базовые параметры
Длительность окна FFT. Оптимальный размер окна — 0,5–2 секунды. Почему:

при меньшем окне снижается частотное разрешение (не получится чётко выделить гармоники дрона);

при большем окне теряется временное разрешение (можно пропустить короткий звук дрона или не заметить изменение оборотов пропеллеров).

Частота обновления. Чем чаще — тем лучше реакция системы, но выше нагрузка на процессор. Баланс:

5 Гц (каждые 200 мс) — минимум для надёжного обнаружения;

10–20 Гц (50–100 мс) — оптимально для отслеживания манёвров дрона (ускорение, торможение, изменение высоты).

Что такое частота дискретизации
Частота дискретизации (sample rate) — это количество отсчётов звука, взятых за одну секунду. Измеряется в герцах (Гц). Например:

44 100 Гц — стандарт для CD‑качества: 44 100 отсчётов в секунду;

48 000 Гц — стандарт для видео.

Чем выше частота дискретизации, тем точнее цифровая копия аналогового звука, особенно высокочастотных компонентов.

1. Центр масс (Center of Mass)
Показывает, где сосредоточена основная энергия спектра. Для дронов характерно значение 200–800 Гц.

javascript
function calculateCenterOfMass(dataArray, sampleRate, fftSize) {
  const binWidth = sampleRate / fftSize;
  let numerator = 0;
  let denominator = 0;

  for (let i = 0; i < dataArray.length; i++) {
    const frequency = i * binWidth;
    const amplitude = dataArray[i];

    numerator += frequency * amplitude;
    denominator += amplitude;
  }

  return denominator > 0 ? numerator / denominator : 0;
}
Интерпретация:

если результат в диапазоне 200–800 Гц — вероятно, дрон;

если < 100 Гц — скорее шум транспорта;

если > 1500 Гц — высокочастотный шум (ветер, шипение).

2. Спектральный поток (Spectral Flux)
Измеряет, насколько быстро меняется спектр между кадрами. Дроны создают пульсирующий звук из‑за изменения оборотов пропеллеров.

javascript
let previousSpectrum = null;

function calculateSpectralFlux(currentDataArray) {
  if (!previousSpectrum) {
    previousSpectrum = [...currentDataArray];
    return 0; // нет данных для сравнения
  }

  let flux = 0;
  for (let i = 0; i < currentDataArray.length; i++) {
    const diff = currentDataArray[i] - previousSpectrum[i];
    flux += Math.pow(diff, 2); // квадрат разности
  }
  flux = Math.sqrt(flux); // корень суммы квадратов

  // обновляем предыдущий спектр
  previousSpectrum = [...currentDataArray];

  return flux;
}
Интерпретация:

высокий спектральный поток (> 50–100) — признак дрона (пульсации);

низкий поток (< 20) — стационарный шум (ветер, гул).

3. Громкость (Loudness / RMS)
Среднеквадратичное значение амплитуды — общая энергия звука. Дроны обычно громче фонового шума.

javascript
function calculateRMS(dataArray) {
  let sumSquares = 0;
  for (let i = 0; i < dataArray.length; i++) {
    sumSquares += Math.pow(dataArray[i], 2);
  }
  return Math.sqrt(sumSquares / dataArray.length);
}
Интерпретация:

RMS > 30–40 — сигнал выделяется на фоне шума;

RMS < 20 — слабый сигнал (возможно, нет дрона).

Перед FFT звук нужно подготовить. Библиотеки предоставляют готовые функции:

Без библиотек вы будете писать эти функции самостоятельно.

3. Расширенные метрики
Помимо центра масс, потока и RMS, для надёжного обнаружения дронов нужны дополнительные признаки:

Mel‑частотные кепстральные коэффициенты (MFCC) — имитируют восприятие звука человеческим ухом, устойчивы к шуму;

спектральная энтропия — измеряет «хаотичность» спектра (дроны дают более упорядоченный сигнал, чем ветер);

кромпаграммы — выделяют гармоническую структуру (характерно для пропеллеров);

временные признаки (zero‑crossing rate, envelope).

Реализация этих метрик с нуля займёт десятки часов.

function calculateCenterOfMass(spectrum, sampleRate) {
  const binWidth = sampleRate / 2048;
  let numerator = 0;
  let denominator = 0;

  for (let i = 0; i < spectrum.length; i++) {
    const frequency = i * binWidth;
    const amplitude = spectrum[i];
    numerator += frequency * amplitude;
    denominator += amplitude;
  }
  return denominator > 0 ? numerator / denominator : 0;
}

function calculateSpectralFlux(currentSpectrum) {
  if (!window.previousSpectrum) {
    window.previousSpectrum = [...currentSpectrum];
    return 0;
  }

  let flux = 0;
  for (let i = 0; i < currentSpectrum.length; i++) {
    const diff = currentSpectrum[i] - window.previousSpectrum[i];
    flux += Math.pow(diff, 2);
  }
  flux = Math.sqrt(flux);
  window.previousSpectrum = [...currentSpectrum];
  return flux;
}

function calculateRMS(spectrum) {
  let sumSquares = 0;
  for (let i = 0; i < spectrum.length; i++) {
    sumSquares += Math.pow(spectrum[i], 2);
  }
  return Math.sqrt(sumSquares / spectrum.length);
}

// 4. Функция анализа с использованием fft.js
function analyzeAudioWithFFT() {
  // Получаем временные данные с микрофона
  const timeData = new Float32Array(analyser.fftSize);
  analyser.getFloatTimeDomainData(timeData);

  // Выполняем FFT через библиотеку
  const complexSpectrum = fft.createComplexArray();
  fft.realTransform(complexSpectrum, timeData);

  // Преобразуем комплексные числа в амплитуды
  const amplitudes = new Float32Array(bufferLength);
  for (let i = 0; i < bufferLength; i++) {
    const re = complexSpectrum[2 * i];
    const im = complexSpectrum[2 * i + 1];
    amplitudes[i] = Math.sqrt(re * re + im * im);
  }

  // Вычисляем метрики
  const centerOfMass = calculateCenterOfMass(amplitudes, audioContext.sampleRate);
  const spectralFlux = calculateSpectralFlux(amplitudes);
  const rms = calculateRMS(amplitudes);

  const isDrone =
    centerOfMass >= 200 && centerOfMass <= 800 &&
    spectralFlux > 50 &&

    amplitudes[i] = Math.sqrt(re * re + im * im); // Амплитуда = |комплексное число|
  }

  const filteredAmplitudes = applyFrequencyFilter(amplitudes, audioContext.sampleRate, 2048);

  // 5. Вычисляем метрики на отфильтрованных данных
  const centerOfMass = calculateCenterOfMass(filteredAmplitudes, audioContext.sampleRate);
  const spectralFlux = calculateSpectralFlux(filteredAmplitudes);
  const rms = calculateRMS(filteredAmplitudes);

  // 6. Добавляем в буфер для усреднения
  addToBuffer('centerOfMass', centerOfMass);
  addToBuffer('spectralFlux', spectralFlux);
  addToBuffer('rms', rms);

  // 7. Получаем усреднённые значения
  const avgCenterOfMass = calculateAverage('centerOfMass');
  const avgSpectralFlux = calculateAverage('spectralFlux');
  const avgRMS = calculateAverage('rms');

  // 8. Логика обнаружения
  const isDrone =
    avgCenterOfMass >= 200 && avgCenterOfMass <= 800 &&
    avgSpectralFlux > 40 &&
    avgRMS > 25;

  console.log({
    'Центр масс (ср.)': avgCenterOfMass.toFixed(1) + ' Hz',
    'Поток (ср.)': avgSpectralFlux.toFixed(1),
    'Громкость (ср.)': avgRMS.toFixed(1),
    'Дрон': isDrone
  });

Физический смысл амплитуды
Амплитуда звуковой волны — это мера отклонения давления воздуха от нормального атмосферного давления в процессе распространения волны. Представьте:

Тишина: давление воздуха стабильно (например, 101 325 Па).

Звук: появляются зоны сжатия (давление выше нормы) и разрежения (давление ниже нормы).

Амплитуда — насколько сильно давление отклоняется от нормы (в Паскалях).

Чем больше амплитуда, тем громче звук.

Как волна превращается в числа
При записи звука микрофон преобразует колебания давления в электрический сигнал (напряжение). Затем аналого‑цифровой преобразователь (АЦП) делает следующее:

Результат — массив чисел, где каждое число отражает мгновенное значение сигнала в определённый момент времени.

Я анализирую не 50 мс звука, я анализирую 2 секунды. Просто я нарезаю замеры на фреймы по 50 мс. Это не слишком длинные отрезки для измерений?
Алиса
Длительность фреймов в 50 мс для анализа звука может быть достаточной или недостаточной в зависимости от конкретных задач и характеристик сигнала. Рассмотрим ключевые аспекты.

Когда 50 мс достаточно
Анализ высокочастотных компонентов. Для частот выше нескольких кГц период колебаний составляет миллисекунды или доли миллисекунды. В 50 мс укладывается несколько периодов таких колебаний, что позволяет оценить их амплитуду и присутствие. Например, для частоты 10 кГц период составляет 0,1 мс, и в 50 мс уместится 500 периодов.
Кратковременные события. Если анализируются импульсные или быстро меняющиеся звуки (например, щелчки, резкие переходы), короткие фреймы помогают уловить динамику без сглаживания резких изменений.
Спектральный анализ с высоким разрешением по времени. Методы, такие как кратковременное преобразование Фурье (STFT), используют короткие окна для баланса между временным и частотным разрешением. В этом случае 50 мс может быть приемлемым компромиссом.
Когда 50 мс может быть недостаточно
Низкочастотные компоненты. Для низких частот (например, 50–100 Гц) период колебаний составляет 20–10 мс. В 50 мс укладывается лишь 2–5 периодов, что может быть недостаточно для точного определения частоты и амплитуды. Например, для точного измерения частоты 50 Гц желательно анализировать сигнал длительностью не менее нескольких периодов, чтобы уменьшить погрешность.
Периодические сигналы с длинным периодом. Если в звуке есть периодические компоненты с периодом больше 50 мс (например, некоторые низкочастотные вибрации или ритмические структуры), один фрейм не охватит полный цикл, что затруднит их идентификацию.
Медленные изменения сигнала. Если звук постепенно меняется (например, плавное нарастание или затухание), короткие фреймы могут не уловить эти изменения полностью, что приведёт к потере информации о динамике сигнала.
Анализ сложных спектральных структур. В случаях, когда важно детально изучить взаимодействие нескольких низкочастотных компонентов или их гармоник, более длинные фреймы обеспечат лучшее разрешение и точность анализа.
Рекомендации
Учитывайте диапазон частот. Если в сигнале преобладают высокие частоты, 50 мс может быть достаточно. Для низкочастотных компонентов стоит увеличить длительность фреймов.
Анализируйте характер сигнала. Для импульсных или быстро меняющихся звуков короткие фреймы оправданы. Для медленно меняющихся или периодических сигналов лучше использовать более длинные отрезки.
Используйте перекрытие фреймов. Если есть риск потери информации из-за коротких фреймов, можно применять перекрытие (например, 50% перекрытия), чтобы улучшить временную локализацию событий.
Экспериментируйте с длительностью фреймов. В зависимости от задачи можно провести серию экспериментов с разными длительностями фреймов и выбрать оптимальную.
В вашем случае, если вы анализируете 2 секунды звука, нарезая его на фреймы по 50 мс, это даст 40 фреймов. Такой подход может быть адекватным для многих задач, но стоит учитывать ограничения, связанные с низкочастотными компонентами и медленными изменениями сигнала. Если эти аспекты критичны для вашего анализа, рассмотрите увеличение длительности фреймов или использование адаптивных методов, которые автоматически корректируют размер окна в зависимости от характеристик сигнала.

Приоритет 2: Добавление новых метрик
1. Спектральный центроид (Spectral Centroid)
Показывает «яркость» звука — дрон обычно имеет умеренную яркость.

javascript
function calculateSpectralCentroid(spectrum, sampleRate) {
  const binWidth = sampleRate / spectrum.length;
  let numerator = 0;
  let denominator = 0;

  for (let i = 0; i < spectrum.length; i++) {
    const frequency = i * binWidth;
    const amplitude = spectrum[i];
    numerator += frequency * amplitude;
    denominator += amplitude;
  }
  return denominator > 0 ? numerator / denominator : 0;
}
2. Спектральная энтропия (Spectral Entropy)
Измеряет «хаотичность» спектра — у дрона она ниже, чем у ветра.

javascript
function calculateSpectralEntropy(spectrum) {
  let sum = spectrum.reduce((acc, val) => acc + val, 0);
  let entropy = 0;

  for (const amp of spectrum) {
    if (amp > 0) {
      const p = amp / sum;
      entropy -= p * Math.log2(p);
    }
  }
  return entropy;
}
